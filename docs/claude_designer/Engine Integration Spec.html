01
Premise & guarantee
Read first

This spec binds four previously-shipped UI artifacts to the concrete engineering work still in flight (P2-T9 → T12, P3-T1 → T7). The goal is one-directional: every visual in the HTML artifacts can be turned into running code — the HTML is the design contract, this document is the execution plan.

01UI Analysis · 16 findings 02Tokens · logos · components 0339-icon sprite 04Editor reference layout
Claude Code can ship every pixel — but only if he treats the HTML as the visual contract, not as code to rewrite. The failure mode is always the same: he sees #e8b923 and renames it --warning-400; or redraws an SVG from memory; or "improves" the grid. All three kill brand coherence.

Architecture is already solid: studio-store.js (observable), studio-preview.js (WebCodecs + LRU), studio-timeline.js (animation-timeline-js), studio-audio.js (wavesurfer-multitrack), studio-beatmarkers.js. The remaining P2 tasks (T9/T10/T11/T12) and Plan 3 need design binding — that's this document.

02
Visual contract
Non-negotiables

Five rules Claude Code must honor. If any one breaks, the cockpit looks generic.

Rule 1 · Tokens ship verbatim

Copy the :root { --p-* } block from Pantheon v3 Direction Spec into static/css/tokens.css. Names stay. Values stay. No semantic renames. No shade generation. If a new value is needed, add it; do not replace existing ones.

Rule 2 · Icons ship as sprite

All 39 <symbol> blocks from Pantheon Icon Set + Pantheon Editor v2 go into static/icons/pantheon.svg. Reference with <svg><use href="/static/icons/pantheon.svg#i-rg"/></svg>. One wrapper component: <Icon name="i-rg" />. Never redraw.

Rule 3 · Grids ship as CSS

The editor's .app grid (grid-template-columns: 260px 1fr 300px; grid-template-rows: 1fr 340px 22px) and the four-track timeline layout are load-bearing. Copy the CSS blocks; make children React components; the grid skeleton does not move.

Rule 4 · Fonts

Bebas Neue for display (section headers, timecodes, weapon ammo, playhead time). IBM Plex Mono for data. No third family. System fonts are the fallback only.

Rule 5 · The four load-bearing "polish" elements are P0

(a) CRT scanline + vignette overlay, (b) gold seam markers between clips, (c) event pins row above the ruler, (d) playhead with gold arrow-head. These read as optional polish; they are not. They are the brand.

06
DM_73 parser spec
P3-T6 deliverable

Quake Live demo format, protocol 91. File is a stream of messages, each with a sequence number + delta-compressed entity state + serverCommands (incl. print messages that encode the kill feed). Parsing is a solved problem — libuberdemotools (already vendored per P1-T5) handles 99% of it. The scaffold is a thin C++17 wrapper that emits our canonical struct.

Canonical struct — DemoStream

struct Snapshot {           // per-tick, ~25Hz (40ms)
  uint32_t  server_time_ms;
  uint8_t   player_count;
  Player    players[MAX_CLIENTS];  // pos, angles, hp, armor, weapon, ammo
  Entity    entities[MAX_ENTITIES]; // projectiles, items, gibs
};

struct Event {              // atomic — frag, pickup, mod_change, etc
  uint32_t   server_time_ms;
  EventKind  kind;             // see table below
  uint8_t    attacker_id;
  uint8_t    target_id;
  Weapon     weapon;
  Modifier   modifier;          // HEADSHOT, TELEFRAG, SUICIDE
};

struct DemoStream {
  DemoMeta            meta;    // map, gametype, duration, ql_version
  std::vector<Snapshot> snaps;
  std::vector<Event>    events;  // all events, time-sorted
  std::vector<ChatLine> chat;    // say/say_team prints
};

07
Manual extraction — Scout workflow
human-in-the-loop
Direction change. The previous auto-tier heuristic (≥3 frags = T1, RG single = T2, etc.) is removed. Tiering was arbitrary and fought the editor's judgment. Replaced with a manual Scout page where the human marks clips, tags them, and flags the demo as reviewed. No ML. No automatic tiering.

The Scout page sits between Archive and Editor. One demo in → N hand-picked clips out → demo flagged clean when the human confirms they've extracted everything worth keeping.

Auto-name helper (suggestion only, not required)

// When the user marks IN/OUT, pre-fill the name field from events inside [t_in, t_out]:
"{dominant_weapon} {event_hint} {map}"
// Examples — user can overwrite freely:
"rail 4x asylum"         // 4 FRAG events, RG dominant
"rocket telefrag dm6"    // TELEFRAG event present
"lg duel campgrounds"    // LG dominant, no special mod
Why manual beats auto. The original auto-tier treated "3 frags in a window" the same as "cross-map railshot while airborne" — both T1. The editor always re-tiered anyway, so the suggestion was noise. The event-pin coverage indicator in the ribbon gives the human the exact information auto-extraction tried to give: have I mined this demo? — without pretending to judge the frag.

Scout output is the same JSON shape as GET /api/studio/clips. Clips committed in Scout show up in the Library panel immediately; no re-import step.

Live-link protocol
engine ⇄ UI

Bi-directional. Engine publishes state, UI publishes intent. Transport is WebSocket to ws://localhost:9731/live — simplest to debug, plenty of bandwidth for 25-60Hz state frames. Claude Code may promote to shared memory later if latency budget demands it; the schema is wire-agnostic.

Engine → UI

25Hz state frames + instant event messages.

{"type":"event", "t":47417,
 "kind":"FRAG","weapon":"RG",
 "target":"sarge"}

UI → Engine

Commands. Acks on the same connection tagged resp.

{"type":"cmd","id":"c42",
 "op":"seek","t":47000}

{"type":"cmd","id":"c43",
 "op":"fx","effect":"crt","on":true,"strength":45}

{"type":"cmd","id":"c44",

Reconnection

If UI loses socket, show CAPTURE chip in the nav turn amber (token --p-amber) and auto-retry every 1s. State frames that arrive while disconnected are discarded; events are buffered engine-side up to 30s.

Why 9731? Nothing claims it on common OSes and it's easy to recognize in netstat. Port is configurable via CS_LIVE_PORT env var, mirroring the REPO_ROOT / CS_ROOT convention from P1-T2.

The "best of all" question

You said: unify all the engines keeping the best of all. Concrete map of what to pull from each, targeted at movie-making (not competitive play).

The assimilation order

  1. Start from ioquake3 master.
  2. Port QL weapon/physics constants and dm_73 parser (libuberdemotools vendored).
  3. Merge q3mme's capture + motion-blur pipeline into cl_cin.c + cl_avi.c.
  4. Port CPMA multiview + freecam into cgame.
  5. Port Wolfcam cam-paths and demo browser as an optional module (engine/wolfcam/).
  6. Remove multiplayer netcode paths not needed for solo demo playback (saves binary size and surface area).
License compatibility — ioq3 is GPL-2; libuberdemotools is permissive; q3mme is GPL-2 (compatible); Wolfcam is GPL-2 (compatible). QL client is closed; we only port constants and use the demo format, both of which are fair. Theatre Studio is AGPL-3 but used as a local tool, not shipped in a server — per the CLAUDE.md exception, that's fine.
11
Where Claude Code drifts
Guardrails

Drift 1 · Token renaming

Symptom: --p-gold becomes --warning-400 or --accent. Two refactors later, someone replaces the "warning" color.

Fix: tokens.css is written once from the v3 Spec. Every subsequent PR that touches tokens requires the literal block unchanged. Add a pre-commit grep: if --p-gold disappears, fail.

Drift 2 · Icon redraws

Symptom: A new feature needs the rail icon. Claude Code writes fresh SVG from memory; geometry drifts. Now there are two rails.

Fix: single pantheon.svg sprite under static/icons/. <Icon/> component is the only way to render one. Lint rule: ban inline <svg> outside the sprite file.

Drift 3 · "Polish" deletion

Symptom: CRT overlay, gold seam markers, event pin row, playhead arrow — all marked low-priority and slipped to "phase 2". They never come back.

Fix: treat each as a P0 acceptance criterion on the corresponding task (T6 timeline, T11 inspector, T12 layout). Do not accept a PR without them.

Drift 4 · Library theme leaks

Symptom: LiteGraph, Tweakpane, Multitrack ship with their default themes (Tweakpane's blue-gray, LiteGraph's teal + dim-gray). Instant visual dissonance.

Fix: before any use, load the override stylesheets (§04 for each). Gate: screenshot test per page; if pixel-diff shows non-Pantheon color in the panel region, fail.

Drift 5 · Grid "improvements"

Symptom: "Let's make the inspector collapsible", "let's make the timeline bigger", "let's use flexbox instead". Density collapses.

Fix: the .app grid is a locked component. Collapsible panels are fine as future work but do not change the default template. Reference the Editor v2 screenshot on every PR that touches layout.
