Brief
Context
Mission

Unify four divergent design systems (style.css, pantheon.css, editor.css, cinema.css) into a single Pantheon v3 system. Apply all 16 findings from the inspection report. Re-brand with a Roman Pantheon motif rooted in meaning (sanctuary for all gods, oculus, pediment, columns) while keeping the Nauru-flag color canon (blue + gold).

Non-negotiables

  • ▸ Strict blue + gold palette (Nauru flag canon). Red = error only. Amber = warning.
  • ▸ Single gold: #e8b923 (hybrid — balances Nauru stripe authenticity with on-screen punch).
  • ▸ Bebas Neue for display, IBM Plex Mono for data/body. Drop Inter. Drop Black Ops One.
  • ▸ CRT scanlines off by default, opt-in toggle.
  • ▸ All custom buttons must have :focus-visible outlines.
  • ▸ No hidden aliases. Token --orange is deleted. Use --p-gold or semantic names.

Token naming convention

All tokens prefixed --p- for "pantheon." This prevents collision with any third-party CSS and makes the system self-documenting. Old names (--orange, --accent, --bg-0) are removed entirely — no compat shims.

Canonical

Gold — the primary accent

Critical semantic rule

Gold is brand + primary action only. Not OK-state, not tier-1, not "drop section." Those need their own signal. T1 can reference gold via border-color, but the solid fill is reserved for brand/primary. This is the single most important fix in v3.

Typography
2 faces · 5 sizes

Bebas Neue for display (logos, part-nav, panel titles). IBM Plex Mono for everything else. Black Ops One and Inter are removed entirely. Don't load three display faces — they fight.

Density rule

No text below 9px carries semantic meaning. 7–8px sizes from v2 are eliminated. If a label is worth putting on screen, it's worth 10px minimum. Ruler/tick labels on data visualizations can be 9px because context carries meaning.

Tracking rule

Letter-spacing scales inversely with string length. Short caps pills: .22em. 5–10 chars: .14em. 10+ chars: .08em. Never exceed .24em on strings longer than 4 characters.

03
Logo explorations
6 directions · pick 1–2

All rooted in the Roman Pantheon: sanctuary for all gods, the oculus that lets the sky in, the pediment, the columns, the coffered dome. Each mark holds at 16px favicon size and at full header scale. Gold on blue-deep or flat — never gradient. Flag canonical #FFC61E reserved for direction 04.

01 · OculusThe open eye of the dome. Radial coffers, glowing center. Literal Pantheon architecture at its most iconic moment.
02 · PedimentTemple facade — triangle over columns. Clan mark carved into the tympanum. Reads as stability, authority, permanence.
03 · Coffer DomeHalf-dome with the coffered interior geometry. Wordmark sits on the dome base. Works as a full lockup.
04 · Flag + OculusNauru flag cropped as the banner field, oculus punched through the blue as the clan's "sky." Literal, canonical, unambiguous.
05 · pTn MonogramThe clan handle as architectural letterforms — big P, big T, lowercase n as a "pillar counterpart." Unique ligature.
06 · All-Gods Ring7 niches (the real Pantheon has seven alcoves) around a central oculus. Literal floorplan abstraction. Most conceptual of the six.
Recommendation

Primary: 02 Pediment for full lockup in headers — most recognizable architectural reference, scales well, holds the pTn mark. Secondary: 01 Oculus as favicon/mark-only — works at 16px, unique silhouette. Skip 04 unless a Nauru-flag moment is specifically wanted (it competes with the other marks for identity).

Patterns
Findings 5–11

Empty states (finding #8)

Every panel that can be empty needs an empty state. Reuse the Compare drop-zone pattern. Dashed border, icon, prompt text, optional JSON-schema hint underneath. 

<div class="p-empty">
  <div class="p-empty-icon">⊞</div>
  <p class="p-empty-title">NO FLOW PLAN LOADED</p>
  <p class="p-empty-hint">drop flow_plan.json or serve from repo root</p>
  <input type="file" accept=".json" class="p-empty-drop" />
</div>

Keyboard scrubbing (finding #7)

Replace the div-based scrubber with a styled native range input. Bind shortcuts:

CRT toggle (finding #12)

Off by default. Toggle lives in a settings popover off the status area. Persisted to localStorage.pantheon.crt.

// boot
if (localStorage.getItem('pantheon.crt') === '1') document.body.classList.add('crt');

// toggle
function toggleCRT() {
  document.body.classList.toggle('crt');
  localStorage.setItem('pantheon.crt', document.body.classList.contains('crt') ? '1' : '0');
}

Resizable panels (finding #5)

Replace hard-coded grid-template-columns: 38% 1fr 24% with CSS grid named areas and JS-driven drag handles that write into a CSS custom property --panels-split. Persist splits to localStorage.pantheon.layout. Min-widths: 280px compare, 400px flow, 200px events.

Migration plan
3 waves

Wave 1 — Foundations (same day)

  1. Create creative_suite/frontend/tokens.css from the canonical block in §01.
  2. Import tokens.css as the first <link> on all 4 HTML files.
  3. Global find-replace in style.css + pantheon.css + editor.css:
    • --orange → --p-gold (all variants)
    • --accent → --p-gold
    • --bg-0..3 → --p-bg / --p-surface / --p-surface2
    • #f5a623 / #f5c518 / #fbbf24 → var(--p-gold)
    • #0a0a0a / #0a0a0c / #040610 → var(--p-bg)
  4. Delete the --orange: var(--gold) shim at pantheon.css:75–77.
  5. Raise --text-mute from #2a3060 / #333 to #5a6890.
  6. Add font-family: var(--p-font-data) to body in editor.css (drop Inter).
  7. Remove all @import for Black Ops One. Keep only Bebas Neue + IBM Plex Mono.

Wave 2 — Structure (next day)

  1. Build the global nav strip component (see §04 mockup). Add to all 4 HTML files as the first child of <body>.
  2. Add :focus-visible rules to all button classes. Rule: 1px dashed var(--p-gold), 2px offset.
  3. Add aria-label to icon-only buttons (▶ PLAY, ⇌ SYNC, +/− ZOOM).
  4. Replace div-based scrubber with <input type="range">. Wire keyboard shortcuts (§05).
  5. Add empty-state pattern (.p-empty) to Flow and Events panels.
  6. Move CRT scanlines behind body.crt class. Default off. Add toggle in status area.
  7. Split gold's semantic usage: LED ok → --p-green. Seam markers → white. Warning → --p-amber.

Wave 3 — Polish (day 3)

  1. Replace panel-bar gold-fill with blue-dim + gold-border (see §04 pattern). Less shouty.
  2. Drop one display face — keep Bebas, delete Black Ops One usages.
  3. Reduce letter-spacing per the §02 tracking rule.
  4. Integrate chosen logo (recommended: Pediment full lockup + Oculus favicon).
  5. Add resizable panel dividers (§05).
  6. Add inline progress bar for long-running operations (replace LED-only feedback).
  7. Hide JSON-loader buttons behind dev-mode detection. Show single banner in dev mode.
  8. Remove "v2" from logo-sub. Move version to status area tooltip.
